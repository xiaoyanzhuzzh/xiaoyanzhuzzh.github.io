AWS实战(1)

本期 AWS 实战主要介绍以下内容:

  1. AWS IAM (Identity and Access Management) & MFA (Multi-Factor Authentication)
  2. AWS S3 (Simple Storage Service)

在介绍之前，首先让我们了解（思考）下面的问题：

  1. 什么是 AWS?
    AWS 是亚马逊公司旗下云计算服务平台，面向用户提供包括弹性计算、存储、数据库、应用程序在内的一整套云计算服务，帮助企业降低IT投入成本和维护成本。
  2. 为什么要用 AWS?
    很多人的答案可能是我们所服务的客户要用 AWS，那么其实再深入的思考下去，客户使用 AWS 无非是因为 AWS 的以下诸多优点：
    • 能满足并且实现绝大多数的业务需求: AWS 提供可靠的，基于云的解决方案，以帮助您满足业务需求。
    • 成本低: 节省了我们自己去搭建机房以及服务器的过程，同时 AWS 是基于按使用量收费的，节约 Money。
    • 风险小: AWS 有着遍布全球的 HA (High Availability) 保证, 而且 AWS 整个生态系统非常全面可靠。

IAM (Identity and Access Management)

IAM 使您能够安全地管理对 AWS 服务和资源的访问。
为什么从 IAM 讲起，是因为 IAM 是用户可以访问所有资源的起点。

  1. 可以使用 IAM 创建和管理 AWS 用户(Users) 和组(Groups)，并使用各种权限来允许或拒绝他们对 AWS 资源的访问。
  2. IAM 是 AWS 账户提供的一项功能，不另外收费。

那么提到 IAM，其实必须要提及一下 MFA (Multi-Factor Authentication), 它可以在用户名称和密码之外再额外增加一层保护。启用 MFA 后，登录 AWS 网站的时候，系统将要求他们输入用户名和密码(第一安全要素 – 用户已知)，以及来自其 AWS MFA 设备的身份验证代码(第二安全要素 – 用户已有)，从而对 AWS 账户设置和资源提供更高的安全保护。

自己私下联系的时候可以尝试这两个例子：
例1：使用 IAM 创建 一个 AWS 用户(Users)，管理该用户及其访问权限，并对该用户使用 MFA.
例2：使用 IAM 创建一个 AWS 角色 (Role)，并且管理其权限.

看完上面的例子，我们可能要思考这个问题：IAM 中 Groups, Users, Roles 的区别是什么?

  • AWS User 是一个能被 AWS 服务和应用程序识别的独一无二的身份, 它拥有永久性长期凭证 (permanent long-term credentials) 并且可以直接跟 AWS 服务交互。

  • AWS Group 是多位 IAM User 的 集合 (A group is a collection of IAM users)。IAM Group主要是为了方便管理一组 IAM 用户的同一组权限。

  • AWS Role 是 IAM 实体，它可以为提交 AWS 服务请求定义一组权限。英文可能更好理解一点儿 (An IAM role is an IAM entity that defines a set of permissions for making AWS service requests, IAM roles allow you to delegate access with defined permissions to trusted entities without having to share long-term access keys. IAM roles cannot make direct requests to AWS services. they are meant to be assumed by authorized entities, such as IAM users, applications, or AWS services such as EC2. Use IAM roles to delegate access within or between AWS accounts.

S3 (Simple Storage Service)

从它的名字可以猜到这个服务是有关存储的，它其实是专为从任意位置存储和检索任意数量的数据而构建的对象存储服务。这些数据可以是任意类型，来源于任意位置。S3的特点：

  • S3 旨在提供 99.999999999% 的持久性, 可存储每个行业数百万个应用程序数据。
  • S3 提供了全面的安全性和合规性功能，甚至可满足最严格的法规要求。用户可以灵活的管理数据，以实现成本优化，访问控制和合规性。
  • S3 提供实时查询功能，可以在S3对静态数据进行强大的分析。
  • S3 是最受支持的云存储服务。

总之S3在存储方面有着很大的优势, 可以使用 S3 做一下事情：

  • 备份和恢复
  • 数据存档
  • datalake 和大数据分析
  • 灾难恢复（跨地区复制（CRR））
  • host static websites

下面是 S3 常用到的命令：

aws s3 ls s3://zhang.trainee // 查看zhang.trainee这个bucket
aws s3 rb s3://zhang.trainee –force // 删除 zhang.trainee这个bucket

aws s3 mb s3://zhang.trainee // 新建一个叫做zhang.trainee的bucket
aws s3 cp local.file s3://zhang.trainee // 将本地文件拷贝到hang.trainee这个bucket上
