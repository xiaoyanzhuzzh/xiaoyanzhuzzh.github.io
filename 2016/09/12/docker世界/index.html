Docker Machine

Docker Machine是一种提供在不同的虚拟主机上安装Docker引擎的工具,并且可以通过docker-machine系列命令管理这些主机. 我们可以使用Machine, 在Mac或者Windows Box, 或者某一网络, 数据中心, AWS等云服务中创建Docker Hosts.

使用docker-machine命令, 可以启动, 检查, 停止和重启被管理的虚拟主机, 并且可以升级Docker client和Docker Daemon, 配置一个Docker Client跟你的虚拟主机通信.这样我们可以在某一个running的Host中直接使用docker命令.

eg. 下面这一命令就指向了一个名叫default的host.

docker-machine env default

NOTE: 在Docker 1.12之前, Docker Machine是在Mac和Windows上运行Docker的唯一方式. Docker v1.12之后, 在Mac和Windows上运行Docker的最佳方式就是Docker for Mac 和 Docker for Windows.

Docker Machine的使用场景

  1. 在an old desktop system上运行docker
  2. Provision Docker hosts on remote systems

Docker Engine VS Docker Machine

当人们提及docker的时候一般说的都是Docker Engine. 那么问题又来了, Docker Engine又是什么呢? 其实它就是一个有Docker Daemon, RESTful API和Docker Client组成的Client-Server Application. 实际的沟通流程就是Docker Client通过Restful API跟Docker Daemon保持通信.
