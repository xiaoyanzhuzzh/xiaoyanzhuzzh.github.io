解决ios滑动卡顿的问题

背景介绍

最近的项目是一个混合APP（hybird App）开发模式，它可以理解为通过 Web 网络技术（如 HTML，CSS 和 JavaScript）与 Native 相结合的混合移动应用程序。

H5用于大体界面的编写，如：需要一些基本的输入框、单选按钮、普通按钮、以及下拉选择框等。CSS3则是主要用于对整体界面细节化的修饰。比如：一个普通按钮，输入框边角默认是直角，那我们可以用CSS来改变其形状。还可以用来设置不同的样式。JS主要是要跟服务端打交道，实现数据交互。JS中的数据交互，主要以JSON格式跟XML格式这两种格式实现。总体来说，H5+CSS3负责界面的搭建，JS负责数据的交互。

在这个项目的开发过程中，我被IOS滑动卡顿的问题长期困扰。基本上事情是这样的，当你对某个div或模块使用了overflow: scroll属性，在iOS系统的手机上浏览时，则会出现明显的卡顿现象（滑动效果就是滑一下滚动一下，而不是我们想要的那种滑一下，可以滚动一段距离的效果）。注意此问题只在IOS系统中有，Android系统没有这个问题，可以自己写一个简单的HTML验证一下。

那么解决这个滑动问题就是使用这个神奇的属性-webkit-overflow-scrolling并将其值设为touch，能够解决问题的原因是因为这个属性启用了硬件加速特性，所以滑动很流畅。不清楚的话可以查看MDN的信息解释https://developer.mozilla.org/en-US/docs/Web/CSS/-webkit-overflow-scrolling.

-webkit-overflow-scrolling引起的其他的问题

虽然-webkit-overflow-scrolling确实启用了硬件加速特性，滑动也变得流畅，但是它带了一另外一个很tricky的问题，就是在使用的这个属性的页面会出现偶尔卡住或者不能滑动的问题。
常见的例子有(我做的项目中经常出现)：

  1. 当滑动到页面底部，先往下滑动在网上滑动时，发现页面卡在底部位置卡住不动
  2. 当一个弹出层使用了这个属性时，我们滑动弹出层意外的其他区域，再在弹出层的区域滑动时，发现页面卡住不动
  3. 当手指在斜着滑动页面时（一支手滑动通常很容易斜着滑），也会发现页面卡住不动

以上都可以通过时间解决问题，就是我们等待几秒钟（2s-5s），再去滑动期望滑动区域时方可滑动流畅，但是这样的解释似乎对于用户来说很难接受，确实除了技术人员我们也很难要求其他专业领域的人员去理解。

对于上述引起的问题，推荐的解决方案

上述问题，已经有很多前辈发现了并提供了一些解决方案，我也在项目中尝试了很多，最后发现如果不行费心，直接使用插件可能会解决这个问题，推荐的插件有iScroll，better-scroll，至于用哪一个，大家可以根据自己项目的情况或者个人喜好去选择。
